## 자바 콘솔 게시판

---

## 📌 주요 구현 기능

### 1. 회원 관리 시스템
- **회원가입**: 아이디 중복 체크 및 유효성 검사 로직 적용.
- **로그인/로그아웃**: `static` 변수를 활용해 현재 로그인한 사용자 객체를 참조 및 로그아웃 기능(로그아웃 시 뱐수 초기화).
- **입력 검증(Validation)**: 
  - 아이디(6자 이상), 비밀번호(영문/숫자/특수문자 조합), 이름(숫자 불가) 검증.
  - 정규표현식(Regex)과 `while` 루프를 활용해 올바른 값이 입력될 때까지 반복 유도.

### 2. 게시판 서비스 (CRUD)
- **목표**: 게시글 작성, 전체 목록 조회, 상세 내용 보기, 수정 및 삭제.
- **권한 제어**: 본인이 작성한 글에 대해서만 수정 및 삭제 메뉴가 활성화되도록 설계.
- **조회 최신화**: 리스트 역순 순회를 통해 최신 등록된 글이 상단에 노출되도록 처리.

### 3. 데이터 저장 및 불러오기 (File I/O)
- **객체 직렬화**: `Serializable` 인터페이스를 사용하여 객체 데이터를 `.dat` 파일로 저장.
- **자동 동기화**: 프로그램 시작 시 파일을 읽어 기존 데이터를 복구하고(불러오기), 마지막 글 번호를 기준으로 새 번호 발급.
- **예외 처리**: 파일 존재 여부 확인 및 데이터 유무에 따른 조건부 저장 로직으로 시스템 안정성 확보 (try-catch | try-with-resources 활용).

### 🔍 기술적 고민과 해결 (Troubleshooting)

- **객체 저장 시 발생한 인증 문제**: `Member` 클래스에 `Serializable` 인터페이스를 구현하여 객체 전송이 가능하도록 해결. (에러 발생시 문제되는 객체 던지는 메세지만 표시)
- **불필요한 파일 입출력(I/O) 방지 및 데이터 유실 방지**: 프로그램 종료 시점에 리스트가 비어 있지 않은 경우에만 저장을 수행하도록 (방어적)로직 설계. (시스템 자원 아낌, 덮어쓰기 방지)
- **입력 버퍼 충돌**: `Scanner` 사용 시 발생하는 버퍼 문제를 `nextLine()`을 활용하여 해결.
